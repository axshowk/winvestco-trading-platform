---
description: Automatically stages changes, generates a descriptive commit message, and commits them.
---

1. Check the current status of the repository using `git status`.
2. Stage all changed files using `git add -A`.
3. Review the staged changes using `git diff --cached` to understand what has been modified.
4. Generate a high-quality, descriptive commit message based on the actual changes.
5. Execute the commit using `git commit -m "<message>"`.
6. Inform the user that the changes have been committed successfully.
